1. 자바에서 상속에 대해서 정리해 서술 해주세요.
	상속이란, 어떤 객체가 고유의 기능(멤버변수, 메서드 등)을 유지하면서
	자신이 아닌 다른 객체의 기능을 물려받아 자신의 기능을 확장(extend)시키는 개념이다.
	상위 클래스를 직접적으로 간섭 않고 하위 클래스의 기능을 확장할 수 있기 때문에 구조가 복잡해질 수 있지만 쓰임새에 따라 편리하다.
		ex) 숫자 a와 b를 더하는 기능을 하는 클래스 A가 숫자 a에서 b를 빼는 기능을 가진 클래스 B를 상속받는다면,
			클래스 A는 숫자 a와 b를 더하는 기능과 빼는 기능, 두가지를 가질 수 있게 된다.


2. 오버로딩에 대해서 정리하여 서술해주세요.
	메서드를 호출할 때, 메서드 간에 같은 이름을 이용하더라도 메서드의 매개변수(파라메터)를 기준으로 구분을 지을 수 있게하고
	이에 따라 서로 같은 이름을 가진 메서드가 여럿 있어도, 호출 시 입력받은 인수에 알맞은 파라메터를 가진 메서드를 호출하는 것이 가능하다.
	다만 이때, 이름이 같아도 리턴 데이터 타입이 다르면 오버로딩의 대상이 될 수 없다.
		ex) 어떤 클래스에 임의의 메서드 void a(int aNum) 와 void a() 가 동시에 존재 할 경우,
			메서드 a 호출 시 int형 데이터를 인수로 사용한다면 전자가 호출된다.

	  
3. 오버라이딩에 대해서 정리하여 서술해주세요.
	상위 클래스에 존재하는 메서드를 하위 클래스에서 재정의, 메서드의 내용을 바꿔서 사용할 수 있는 기능이다.
	다만, 서로의 리턴 데이터 타입, 이름, 매개변수가 같아야 하는 것이 조건이다.
		ex) a+b의 출력(println)을 수행하는 void a(int num1, int num2) 메서드를 가진 클래스 A가 있고
			이를 상속받는 하위 클래스 B가 있을때, B에서 void a(int num1, num2) 메서드를 선언한 뒤
			메서드 내용을 a-b의 출력으로 재정의하고 B.a(3,2)를 호출하게 되면, a-b의 출력을 수행한다.
			(이때 메서드 a는 static이라고 가정한다)


4. 생성자와 메서드와 차이점에 대해 정리하여 서술해주세요.
	생성자는 클래스 객체를 새로 생성할 때 자동으로 호출되며, 필수적인 작업이다.
	생성자의 파라미터와 내용에 따라 새클래스 객체 생성될 때 값을 지정해 주거나(초기화), 특정한 명령을 내릴 수 있다.
	또한 어떤 값도 반환하지 않고, 이름이 해당 클래스의 이름과 같다는 점에서 메서드와 차이가 있다.

 - 자바의 특성과 장점을 찾아서 서술해 주세요 -
 1. 많은 사용자와 레퍼런스
  - 비교적 배우기 쉬운 언어의 장점이 있기 때문에 여러 사람들이 JAVA를 사용할 수 있었고(대중성),
  	이로 인해 많은 연구를 통해 안정성을 확보하면서 JAVA를 이용하는 대형 프로젝트가 진행되었고, 많은 참고자료와 오픈소스 API들이 생겨났다
  	
 2. JVM에 의한 높은 운영체제 호환성 (Write Once, Run Anywhere)
  - JAVA로 만들어진 프로그램은 어떤 운영체제에서든 JVM을 통해서 똑같이 작동한다.
  	이는 운영체제 마다 다른 코드를 작성할 필요가 없게 만들기 때문에 생산성면에서 커다란 이점이 있다고 할 수 있다.
  	다만 운영체제마다 미묘한 차이가 있으므로, 특정 운영체제를 대상으로 개발되어 다른 운영체제를 고려하지 않은 프로그램을
  	필요에 타 운영체제에서 실행해야 할 때에는 별도의 테스트가 필요하다.(하지만 JAVA는 다른 언어에 비해 이런 부분들이 적은 편이다.)
  	
 3. 자동 메모리 관리 (가비지 컬렉터)
  - 사용하지 않는 변수, 메서드등을 메모리에서 자동으로 제거해주는 기능인 '가비지 컬렉터'의 존재로 인해
  	JAVA 개발자는 메모리 관리에 크게 신경쓰지 않아도 되기 때문에 생산성의 향상에 큰 도움을 준다.
  	
 4. 강고함
  - JAVA는 안정성 유지를 위해 약간의 자유도를 희생하여 다른 언어에서 지원하는 많은 기능을 제한하고 있다.
  	예를들면, 객체간의 다중 상속의 배제(클래스 재사용을 통한 생산성의 향상과 관리 상의 이점이 있지만 복잡하게 꼬인 프로그램이 될 위험성을 갖고 있다)
  	메모리 직접접근 함수 미지원, 예약어의 오버라이딩 불가, 최대한 많은 오류를 배제하는 컴파일러 등..
